Technology

CoreNuc Home Poker Tournament Director is built with a modern .NET application stack for cross-platform user interfaces, local tournament data, live display features, and reliable release packaging.

Core Application Stack

.NET 10

The application targets .NET 10 and uses modern C# with nullable reference types and implicit using directives enabled.

.NET MAUI

The shared application is built with .NET MAUI so the same tournament workflow can be delivered across desktop and mobile platforms.

C#

C# powers the app logic, tournament rules, data services, settings, receipts, player actions, standings, and display state.

XAML

XAML defines the app screens, modals, templates, resources, styles, and reusable UI layouts.

Platforms

  • Windows WinUI
  • Windows 10 version 1809 or newer
  • Windows 11
  • Android 5.0 or newer
  • iOS 15.0 or newer
  • Mac Catalyst 15.0 or newer
  • x86
  • x64
  • ARM64

Data And State

SQLite

Tournament data is stored locally with SQLite using sqlite-net-pcl and SQLitePCLRaw.bundle_green.

Local Settings

App preferences store display options, receipt choices, sound selections, themes, and other local user settings.

Database Import And Export

The app supports database backup and restore workflows for moving or protecting tournament data.

Application Architecture

MVVM

View models separate screen behavior from UI markup and organize the app around players, chips, prizes, rounds, tables, settings, and games.

CommunityToolkit.Mvvm

The MVVM toolkit provides observable objects and command patterns used throughout the app.

Shared Services

Services handle tournament sounds, receipts, themes, button display modes, TV display state, and app settings.

Live Display Technology

ASP.NET Core

The desktop build can host a lightweight local web server for the TV Browser Display.

HTML, CSS, And JavaScript

The TV Browser Display renders tournament screens in a browser and cycles through selected views such as timer, blinds, players, tables, prizes, and recent actions.

System.Text.Json

JSON is used to publish tournament display state from the app to the local browser display endpoint.

Features Supported By The Stack

  • Tournament setup for players, rounds, buy-ins, rebuys, add-ons, chips, prizes, and tables.
  • Live tournament clock with round advancement, warnings, player actions, bust outs, undo, and final results.
  • League and season standings with score, wins, cashes, knockouts, average finish, paid, won, and net results.
  • Player receipt generation and sharing through platform-supported share features.
  • Customizable sounds and local display preferences.
  • TV browser display for sharing tournament information on another screen on the same home network.

Testing And Release

xUnit

Automated tests use xUnit with the Visual Studio test runner integration.

coverlet

Test coverage support is included through coverlet.collector.

Windows Packaging

The Windows project uses WinUI, MSIX tooling support, application icons, runtime identifiers, and platform-specific packaging settings.
