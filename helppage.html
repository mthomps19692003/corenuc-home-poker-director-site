Step By Step Game Setup

1. Start on the Texas Holdem page. Enter the game name, description, league, season, game type, and any notes that should be saved with the tournament.

2. Add or select the players for this game. Use Players from the Game Setup area, choose the players who are playing, then return to the Texas Holdem page.

3. Set up the blind rounds. Open Rounds, add the tournament levels and breaks, and select the round structure for the current game.

4. Choose the tournament options. Open Tournament Options to select the buy-in, rebuy, and add-on records that apply to this game.

5. Build the chip set. Open Chips, select the chips used by the game, and confirm that the required chip quantities match the number of players.

6. Set the prize payouts. Open Prizes, calculate or enter the payouts, and mark the prizes that should be displayed during the tournament.

7. Seat the players. Open Tables after the player list is final, then assign seats manually, randomly assign seats, or balance active players across tables.

8. Return to the Texas Holdem page and review the Game Setup indicators. Any item that still needs attention can be selected from that area.

9. Save the game. Saving stores the setup so it can be loaded again from Saved Games.

10. Start the Main Display when the tournament is ready. The display runs the clock, shows the current round, warns before cutoff rounds, and lets you record or undo player actions during play.

What The App Does

CoreNuc Tournament Director is used to set up, save, run, and review poker tournament games. The app stores reusable setup items such as players, chip sets, rounds, buy-ins, rebuys, add-ons, prizes, and table seating.

The goal is to keep tournament night organized: build the event once, reuse the setup later, run the clock, track player actions, and keep a record of results without using separate spreadsheets or handwritten notes.

A game is created from those setup items. Once the game is ready, the Main Display can be opened to run the tournament clock, track the current round, record player actions, and finish the game.

Saved results are used by Player Stats to build league and season standings, including points, wins, cashes, knockouts, average finish, and net results.

Most setup pages work the same way: add or edit reusable records, select the records for the current game, then return to the Texas Holdem page to continue building the tournament.

Minimum Requirements

Windows desktop: Windows 10 version 1809 or newer. Windows 11 is recommended for the best install and display experience.

Android phone or tablet: Android 5.0 or newer.

iPhone or iPad: iOS 15.0 or newer.

Phone remote: cross-device phone remote connection is not enabled in this build. The desktop remains the main tournament display and source of truth.

Phone Remote

Phone remote connection is not enabled in this build. The current remote code does not create a network or Bluetooth connection between the desktop app and a phone.

The Main Display can still run the tournament locally from the desktop. Use the desktop controls to record rebuys, add-ons, bust outs, receipts, and player status.

Before phone remote support can be offered to end users, the app needs a real transport layer such as a local network service, Bluetooth service, or another supported device-to-device connection method.

Until that transport is added, the Remote Connection section will show that phone remote connection is not enabled.

TV Browser Display

The TV Browser Display is for showing tournament information on a TV or another computer through a web browser. It runs from the Tournament Display window and publishes a local web page on the host computer.

To get to it, save or load a game, select Start from the Texas Holdem page to open the Tournament Display, then select Customize. In the TV Browser Display area, choose the screens to include in the cycle.

Available TV screens include Tournament Display, Game Details, Timer, Blinds, Players, Tables, Prizes, and Recent Actions. Select only the screens you want shown on the TV.

Set the Delay value to control how many seconds each selected screen stays visible before the browser moves to the next screen. The app allows delays from 5 seconds to 300 seconds.

Select Start TV to start the lightweight web server. The Tournament Display will show a network URL and a local URL. Use the network URL for a TV or another device on the same home network.

On the TV, open its web browser and type the network URL exactly as shown, including http:// and the port number. The TV and the computer running CoreNuc Tournament Director must be on the same Wi-Fi or wired network.

If the TV cannot connect, check that the computer is awake, CoreNuc Tournament Director is still running, the TV is on the same network, and Windows Firewall allows the app to accept local network connections.

Leave the Tournament Display open while the TV browser is in use. The browser refreshes tournament data automatically and cycles through the selected screens until Stop is selected or the Tournament Display is closed.

Basic Workflow

1. Start on the Texas Holdem page and enter the game information.

2. Add the required setup items: players, rounds, buy-ins, chips, prizes, and tables. Add-ons and rebuys are optional, but must be associated with the game before it starts if they will be used.

3. Set late registration rules if needed. Late registration can close when the game starts, after a selected round, after a selected number of minutes, or by whichever selected cutoff happens first.

4. Save the game so the setup can be loaded again later.

5. Start the Main Display when the tournament is ready to begin.

6. Record rebuys, add-ons, bust outs, round changes, and final results while the game is running. Use Undo Last when a bust-out, rebuy, or add-on was recorded by mistake.

7. Review Summary, Settlement, and Player Stats after the tournament is complete.

Optional shortcut: use Settings, Default Game to add a complete starter tournament, then load it from Saved Games and start from a ready example.

Texas Holdem Page

The Texas Holdem page is the main game setup page. Enter the event name, description, league, season, game type, late registration rules, and notes for the tournament.

The Game Setup indicators show whether players, rounds, buy-ins, add-ons, rebuys, chips, prizes, and tables have been selected for the current game.

Selecting an indicator opens the related setup area. This lets the user quickly jump to the missing part of the game setup.

Save stores the current setup. Load brings an existing saved game back to the screen. New starts a fresh game. Reset clears the current screen after confirmation when a loaded game is active.

Start opens the tournament display for the saved game. Summary opens the action summary, and Settlement opens the end-of-game payout and player net-results review.

Game Details opens a read-only dashboard for the saved game. The dashboard shows Players / Seating, Rounds, Prizes, and Notes in scrollable cards so larger games can be reviewed without losing the surrounding layout.

Tooltips are available on the main buttons and explain what common actions will do before they are selected.

Tournament Options

Tournament Options is where buy-ins, add-ons, and rebuys are managed and selected for a game.

Buy-ins define the cost, rake, chips, points, and bounty chip cost for entering the tournament.

Rebuys define when a player can buy back in, how many times it is allowed, the cost, chips received, points, and limits.

Add-ons define extra chip purchases that may be allowed during the tournament, including cost, chips, points, limits, and the last round allowed.

Rebuys and add-ons automatically close after their configured last round. When the Main Display is about to start a round that closes an option, the app shows a warning before continuing.

For a saved game, selecting a buy-in, rebuy, or add-on attaches it to that game. The selected records are then used by the tournament display and player action logs.

Players, Rounds, Chips, Prizes, And Tables

Players lets the user maintain player records and add selected players to the current game. Players added to the game stay visible in the list and can be removed before the tournament starts.

After the tournament starts, adding players follows the saved late registration rules. If late registration is closed, the app prevents the player from being added and explains why.

A player cannot be deleted while attached to a game in progress. Remove or complete the game connection before deleting the player record.

Rounds lets the user create blind levels, break rounds, and saved round structures that can be reused in future games.

Chips lets the user create chip sets and select the chips used for the current game.

Prizes lets the user define prize payouts, awarded places, points, shared amounts, and displayed prize information.

Tables lets the user seat players manually, randomly assign players to seats, clear seating, or balance active players across tables during tournament play.

Chip Inventory And Chip Sets

Chip Inventory tracks the physical chips available to the tournament director. Add each chip color, value, and quantity so the app can compare chip sets against the chips actually on hand.

Chip Setup creates the chip set used by a tournament. Each chip in the set is picked from inventory, then assigned a per-player quantity. The page shows how many chips are required for the current player count and whether inventory is short.

If the correct chip is not already in inventory, choose the add-inventory option from the chip picker while adding or editing a chip set item.

If an inventory chip is already used in a chip set, the app prevents deleting it and warns before editing it. Remove the chip from all chip sets before deleting that inventory record.

Prize Payouts

Prizes can be entered manually or calculated from the current game. Default prize calculation uses the selected buy-in fee and the number of active players bought into the game.

For eight or fewer players, the app calculates the top three prizes. For more than eight players, it calculates the top four prizes. The last prize is always $25.00, and the remaining prize pool is split so first place is the largest payout.

Displayed prizes are attached to the current game. Extra prize rows can remain saved in the setup but be hidden from the tournament display by turning Display off.

After the tournament is complete, Settlement can be used to review gross collected, rake, net prize pool, payouts, and each player's net result.

Tables And Seating

Tables uses the players in the current game to create seating groups. The app supports manual seat assignment, random assignment, and clearing seats when the layout needs to be rebuilt.

Use seating after players have been added to the game. If players are added during late registration, revisit Tables and balance the active players when needed.

Balance Tables works only with active players. Busted players stay in their recorded finish positions while remaining players are redistributed as evenly as possible.

Running A Tournament

Start opens the Main Display in its own window. This lets the tournament display stay active while the main setup window remains available.

The clock does not begin until Start is selected on the Main Display. Pause and Resume control the timer during play.

The display shows the current round, blinds, ante when applicable, upcoming round information, and the active player list.

Next Round moves immediately to the next configured round. When the timer reaches zero, the display advances to the next round.

If the next round closes late registration, rebuys, or add-ons, the app warns before the round starts. Rebuys and add-ons are disabled automatically after their cutoff.

During the game, the user can record player rebuys, add-ons, and bust outs. Bust-out prompts use player nicknames so the confirmation popup stays easy to scan.

Undo Last reverses the latest bust-out, rebuy, or add-on. The Recent Actions area shows the latest tournament activity so mistakes can be spotted quickly.

When only one active player remains, the game is ended and the winner is recorded.

Saved Games And Stats

Saved Games lets the user load a previously saved tournament setup. If that game is saved again, the existing record is updated instead of creating a duplicate.

Player Stats now works as a standings page. It can show a season leaderboard by league and season, with player rank, games played, score, wins, cashes, knockouts, average finish, paid, won, and net.

Overall score is calculated as 5 participation points per game played, plus finish points, plus 10 points for each win, plus 5 points for each cash, plus 2 points for each knockout.

Finish points are based on the tournament field size. The formula is players started minus finish position plus 1, so a winner in a 12-player tournament receives 12 finish points, second place receives 11, and last place receives 1.

Use the start and end date fields, League picker, and Season picker to narrow the standings. Column headers can be selected to sort the results ascending or descending.

Export on Player Stats creates an Excel-readable standings file using the currently displayed results. Date range, league, season, and sort order are included in the export.

Paid includes buy-ins, rebuys, and add-ons. Net is calculated as winnings minus paid. Knockouts come from recorded bust-out credit, and average finish is based on completed player results.

Stats are only as complete as the actions recorded during play, so record buy-ins, rebuys, add-ons, bust outs, and final results from the tournament display whenever possible.

Settings And Sounds

Settings controls the app theme, receipt content, button display mode, and tournament sounds. Sound choices are used by the Main Display for player bust outs, one-minute warnings, and round changes.

The Database area in Settings lets the user export a database backup, import a database backup, or add the built-in Default Game. Export asks where to save the file. Import lets the user either replace the current database or add the imported records to the existing database.

When importing a database backup, Replace removes the current database after a warning and restores the selected file. Add to Existing Database keeps the current data and adds the imported records as new records.

Default Game is optional and only runs when the user chooses it. It adds a ready-to-load starter tournament without deleting existing data.

The Default Game includes 10 sample players, one seated table, a $25 buy-in with 9,000 starting chips, a $15 rebuy for 4,500 chips, a $10 add-on for 3,000 chips, chip values from $5 through $5,000, 15 levels with 3 breaks, and top-three prizes with third place fixed at $25.

After importing the Default Game, open Saved Games, load the new Default Starter Game, review the setup if desired, and start the Main Display when ready.

Main Page Picker Values in Settings lets the user add, edit, or delete the League, Season, and Game Type choices shown on the Texas Holdem page. Items attached to a game in progress cannot be deleted.

Display settings control the theme, button display mode, and the option to disable the system screen saver while the tournament display is running.

Receipt settings let the user choose which tournament details appear when a player receipt is generated, including buy-ins, rebuys, add-ons, and totals.

If a custom sound is added, keep the file available on the device where the tournament is being run.
